# 捷顺ims

## 系统介绍

捷顺ims是一个用于管理商品库存的系统，支持商品的添加、编辑、删除，以及商品的入库和出库操作。系统分为管理员和普通用户两种角色，管理员拥有全部权限，普通用户只能进行入库、出库相关操作。

### 系统信息

- **系统版本**：3.0.1.0
- **系统制作人**：Julissa
- **注意**：本系统不支持用户自行注册账号，所有账号由管理员创建和管理。
- **配套文档**：`agent.md` 提供了完整的代码规范与书写要求，开发新功能时请遵循该文档

### 系统功能

- **商品管理**：添加、编辑、删除商品信息
- **商品入库**：记录商品入库操作，更新库存
- **商品出库**：记录商品出库操作，更新库存
- **入库记录**：查看所有入库记录，支持按商品名称、操作人、日期搜索和筛选
- **出库记录**：查看所有出库记录，支持按商品名称、操作人、日期搜索和筛选
- **用户管理**：（管理员）添加、编辑、删除用户，设置用户权限
- **日志查看**：（管理员）查看系统操作日志
- **数据备份**：（管理员）备份系统数据
- **安全验证**：登录时使用验证码和bcrypt加密，密码错误次数限制，账号封禁功能，提高系统安全性
- **数据导出**：支持将入库/出库记录导出为 `.xls` 文件（UTF-8 BOM，支持中文）
- **分页与搜索**：列表页支持分页（10/15/20/50条/页）和搜索筛选，分页大小记忆到 localStorage
- **库存预警**：库存低于预警值时显示红色背景和白色粗体文字提示

## 系统要求

- PHP 5.6.0 或更高版本
- MySQL 5.6.0 或更高版本
- Web服务器（如 Apache、Nginx 等）

## 安装教程

### 步骤1：准备环境

1. 确保您的服务器已安装 PHP 5.6.0 或更高版本
2. 确保您的服务器已安装 MySQL 5.6.0 或更高版本
3. 确保您的 Web 服务器已配置好并能正常运行

### 步骤2：创建数据库

1. 登录数据库管理工具（如 phpMyAdmin）
2. 创建一个新的数据库（建议使用 `stock_db`）
3. 创建数据库用户并为其分配该数据库的所有权限

### 步骤3：上传文件

1. 将系统文件上传到您的 Web 服务器目录
2. 确保 `config` 目录具有可写权限

### 步骤4：运行安装程序

1. 在浏览器中访问 `https://您的域名/install/install.php`
2. 按照安装向导的提示完成安装过程
3. 安装完成后，系统会自动生成 `config/config.php` 和 `config/install.lock.php` 文件
4. 确保系统文件的权限设置正确，建议设置为 755 或 777
5. 确保系统文件的路径正确，与系统安装时的路径一致
6. 确保系统文件的路径中不包含特殊字符或空格
7. 确保系统文件的路径中不包含中文字符

### 步骤5：登录系统

1. 安装完成后，点击 "进入系统" 按钮
2. 使用默认账号登录：
   - 管理员账号：admin / 123456
3. **安全建议**：安装完成后，建议删除 `install` 目录，以提高系统安全性

## 注意事项

1. 系统安装完成后，`config/config.php` 和 `config/install.lock.php` 文件会被设置为只读权限，以提高安全性
2. 安装完成后，建议删除 `install` 目录，以提高系统安全性
3. 如需重新安装系统，请手动修改 `config/config.php` 和 `config/install.lock.php` 文件的权限后删除它们，然后重新运行安装程序
4. 重新安装会清空所有数据，请在重新安装前备份重要数据
5. 定期使用数据备份功能备份系统数据，以防数据丢失
6. **生产环境建议**：在生产环境部署完成并测试后，建议删除 `pages/products/add.php` 测试文件，以提高系统安全性

## 常见问题

### 1. 安装失败怎么办？

- 检查系统要求的软件和版本是否符合系统要求
- 检查系统文件的路径设置是否正确，与系统安装时的路径一致
- 检查数据库连接信息是否正确
- 检查数据库用户是否具有足够的权限
- 检查 `config` 目录是否具有可写权限

### 2. 页面显示数据库连接失败怎么办？

- 检查数据库服务器是否正在运行
- 检查数据库连接信息是否正确
- 检查数据库用户是否具有访问权限

### 3. 如何修改管理员密码？

- 登录系统后，点击左侧导航栏中的 "修改密码" 链接
- 输入旧密码和新密码，点击 "确认修改" 按钮

### 4. 如何添加新用户？

- 以管理员身份登录系统
- 点击左侧导航栏中的 "用户管理" 链接
- 点击 "添加用户" 按钮，填写用户信息并设置权限

### 5. `agent.md` 是什么？开发时需要遵守吗？

- `agent.md` 是本系统的代码规范与书写要求文档，定义了安全、数据库、页面、日志、权限、导出、安装等方面的统一规范
- 是新增功能、二次开发、代码审查的必备参考文档，所有新增/修改的代码必须遵循该文档中的规范
- 文档中的关键规范包括：必须使用预编译语句防止 SQL 注入、必须使用 `password_hash()`/`password_verify()` 加密密码、所有页面必须定义 `IN_SYSTEM` 常量、输出到 HTML 的内容必须使用 `htmlspecialchars()` 转义、商品操作日志必须使用"品牌-型号"格式等

## 安全提示

- 定期更新系统密码
- 不要使用默认密码
- **密码强度要求**：建议使用至少8位长度的密码，包含大小写字母、数字和特殊字符，避免使用简单的生日、电话号码等容易猜测的密码
- 限制系统访问权限
- 定期备份系统数据
- 定期检查系统日志，发现异常及时处理
- **公网访问建议**：如果系统用于公网访问，建议将程序和数据存储在不同的服务器上，以提高安全性
- **安全建议**：系统部署完成后，建议删除 README.md 和 agent.md 文件，以避免泄露系统信息
- **开发规范**：所有代码必须遵循 `agent.md` 中的规范要求（包括预编译语句、bcrypt 加密、IN\_SYSTEM 常量、htmlspecialchars 输出转义等）
